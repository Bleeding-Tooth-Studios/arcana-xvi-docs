Dialogues

Dialogues are a 2 way communication method triggered by a remoteEvent.

Dialogue NodeTrees are created with Sol_ttu's Node Editor Plugin. It's also a modified version of their dialogue system.

There are three types of Dialogues:

  • No image
  • With image
  • Animated

NodeTrees​

NodeTrees is where the dialogue's node data and behaviors are stored.

Example :

Node Types​

NodePresets is a folder you store on ServerStorage for node presets to appear on the node editor.

  • DialogueRoot -> The start of every dialogueTree
  • Prompt -> Text container
  • ExpressionPrompt -> Text container corresponding to a character expression
  • Response -> A selectable response
  • Command -> Runs a command when a node is being loaded

Creating A Nodetree​

Controls

  • CTRL+D to duplicate a node
  • S to open node presets
  • RMB on a connection to delete it

Running A Dialogue​

Place dialogue in assets folder

Notes​

  • Another dialogue will not run as long as one is currently active

  • When the current prompt node is connected to more than one prompt nodes of the same priority, it will randomize the next one. Otherwise, it selects the node with the highest priority

  • Locked and condition nodes have not been implemented or used yet

  • For moving chaaracters and the talk area, just give it a lot of range since its inconvenient to have it keep disappearing and reappearing

  • Try not to fit too much text into one prompt. Keep it brief

  • Make sure expression prompts are properly declared in faceExpressions.ts, and make sure the Expression field is typed in lowercase and no empty spaces

Dialogue Controls:

  • M1/spacebar to proceed/confirm response
  • mousescroll/up-down arrow keys to select response

These keys are overwritten with ContextActionService as long as a dialogue is active

The dialogueremote accepts 3 parameters, (player, dialogueTree, and dialogueParams)

Examples

Example 1​

//probably replaced later with a d.ts file
const dialogueTrees = ReplicatedStorage.XVIAssets.WaitForChild("DialogueTrees");
const dialogueTreeNoImage = dialogueTrees?.FindFirstChild("TestDialogueNoImage") as Instance;
const dialogueTreeWithImage = dialogueTrees?.FindFirstChild("TestDialogueWithImage") as Instance;
const dialogueTreeWithAnimatedImage = dialogueTrees?.FindFirstChild("TestDialogueWithAnimatedImage") as Instance;

PlayerService.playerJoined.Connect((player) => {

    // Without Image
    if (dialogueTreeNoImage) {
        remoteEvents.dialogueRemote.fire(player, dialogueTreeNoImage, {
            Speaker: "Narrator", //speaker name, also used for setting expressions if using ExpressionPrompt
            Range: 50000, //if you go out of this range. the dialogue will disappear
            Position: new Vector3(0, 5, 0), //the center of the talking range
            hasImage: false, // Explicitly set to false (though it's the default)
            imageId: "", // No image ID needed for No image dialogues
        });
    }

    task.wait(10);
    
    // With Image
    if (dialogueTreeWithImage) {
        remoteEvents.dialogueRemote.fire(player, dialogueTreeWithImage, {
            Speaker: "Bird",
            Range: 50000,
            Position: new Vector3(10, 5, 10),
            hasImage: true,
            imageId: "rbxassetid://96356935951750", //a single string means its a still image
        });
    }

    task.wait(10);

    // With Animated Image;
    if (dialogueTreeWithAnimatedImage) {
        remoteEvents.dialogueRemote.fire(player, dialogueTreeWithAnimatedImage, {
            Speaker: "Malicious Demon",
            Range: 50000,
            Position: new Vector3(10, 5, 10),
            hasImage: true,
            imageId: ["103726754518321", "77047615321639", "131092117371533"], //a list of strings turns it into an animated sprite
            totalFrames: 44,
            frameSpeed: 0.02,
        });
    }
});

Example 2​

You can also trigger dialogues with a touchPart

mikaPart.Touched.Connect((plr) => {
			const player = plr.Parent as Model;
			const Player = Players.FindFirstChild(player.Name) as Player;

			if (mikaDialogue) {
				remoteEvents.dialogueRemote.fire(Player, mikaDialogue, {
					Speaker: `<font color="#2927a8">Mikail</font>`,
					Range: 10,
					Position: new Vector3(315.477, 4.027, -385.617), //position corresponds to model location in workspace
				});
			}
		});

Here you can see that the dialogue is different when the user has already initiated it. On the first time he says "Hey", after that he says "What is it?" These behaviors are controlled by command nodes that change the priority of the prompt nodes.

(yes, you need to do a bit of lua unfortunately)
